### 学んだこと

#### Aurora ストレージの可用性

ストレージは、３つの異なるアベイラビリティゾーンに分散される。

各アベイラビリティゾーンに２つのコピー（計６）が存在する。

ちなみに、RDS の場合は１つのリージョンにのみボリュームが存在するので、可用性はAuroraに劣る。

#### マルチAZ と マルチAZ

* シングルAZ
  * インスタンスの再作成が必要なため、通常10分未満の停止時間が発生（同一AZに自動再生成）
  * AZ障害の場合は、手動作成の必要があります。

* マルチAZ
  * フェールオーバーにより、通常1分未満でデータベース処理再開可能。
  
#### リーダーエンドポイント

リーダーエンドポイントを参照しておけば、ランドロビンでリーダーインスタンスに振り分けることができる。

### Aurora サイズ制限

#### ストレージサイズの制限

```
ストレージサイズの制限
Aurora クラスターボリュームの最大サイズはエンジンバージョンによって異なります。

最大 256 TiB:
Aurora PostgreSQL バージョン

17.5 以降のすべてのバージョン

16.9 以降

15.13 以降

Aurora MySQL バージョン 3.10 (MySQL 8.0.42 と互換) 以降

最大 128 TiB:
以前のすべての Aurora PostgreSQL バージョン

Aurora MySQL バージョン 3 の使用可能なすべてのバージョン、Aurora MySQL バージョン 2 (バージョン 2.09 以降)
```

#### SQL テーブルサイズ制限

```
Aurora MySQL DB クラスターの場合、テーブルの最大サイズは 64 テビバイト (TiB) です。Aurora PostgreSQL DB クラスターの場合、テーブルの最大サイズは 32 テビバイト (TiB) です。大きいテーブルの分割など、テーブル設計のベストプラクティスにしたがうことをお勧めします。
```

1 TiB=240 bytes≈1.0995 TB
