## 2. Windows サーバー環境（PowerShell）から実行する場合

AWS 外のオンプレミスな Windows サーバーや、特定の Windows 端末上のタスクスケジューラから定期実行するケースです。

Windows 環境であれば、OS 標準の暗号化機能を利用した **「Data Protection API (DPAPI)」** や **「Windows 資格情報マネージャー」** を使うのが定石です。

### PowerShell Client Secrets Flow：

事前に、そのスクリプトを実行する **OS ユーザーの権限** でシークレットを暗号化してファイルに書き出しておきます。



```powershell
# 【事前準備：暗号化してファイルに保存】※この処理は1度だけ手動で実行する
"YOUR_COGNITO_CLIENT_SECRET" | ConvertTo-SecureString -AsPlainText -Force | ConvertFrom-SecureString | Out-File "C:\Secure\secret.txt"

```

```powershell
# 【定期実行スクリプト側での読み込み】
# ファイルから暗号化された文字列を読み込み、平文に戻す
$encryptedSecret = Get-Content "C:\Secure\secret.txt"
$secureSecret    = ConvertTo-SecureString $encryptedSecret
$bstr            = [System.Runtime.InteropServices.Marshal]::SecureStringToBSTR($secureSecret)
$plainSecret     = [System.Runtime.InteropServices.Marshal]::PtrToStringAuto($bstr)

# この $plainSecret を使って Cognito にアクセスする

```

### 運用のイメージ：

1. スクリプト内には `$COGNITO_CLIENT_SECRET` という変数名だけを記述しておきます。
2. スクリプトを実行するユーザーのプロフィール（`~/.bashrc` や `~/.bash_profile`）や、サービス管理（`systemd` の環境変数設定）にシークレットを定義します。
3. または、シークレットだけを記述した設定ファイルを別で作成し、**パーミッションを `600`（所有者のみ読み書き可能）** に設定して、スクリプトから読み込ませます。

```bash
# /etc/file_monitor/secret.conf (パーミッションは 600)
COGNITO_CLIENT_SECRET="your_actual_secret_here"

# スクリプト内での読み込み
source /etc/file_monitor/secret.conf

```
