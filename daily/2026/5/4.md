### terraformのバージョンの固定方法について

main.tfの最上部に記述する。

```terraform
# --- ここに書く ---
terraform {
  required_version = "~> 1.5.0"
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
}
```

中規模、大規模は、チーム開発の場合、version.tf に切り出すほうがいいかも


### 削除保護

resource ブロックに 以下を記載する

prevent_destroy は意図しないリソースの再作成を防止する。

DBインスタンス識別子を変更すると、既存DBインスタンスが削除され再作成されることがあり、こういった事故を防ぐ

```
  lifecycle {
    prevent_destroy = true
  }
```
