### マルチステージビルド

ビルドに必要な依存関係や中間生成物をすべてビルド用ステージに隔離し、最終成果物のみを実行用イメージに抽出するビルド方法です。
メリットは、コンテナイメージの容量化少なくなることやビルドコンテナにパッケージマネージャを残さないため攻撃者にツールを利用させないことができる。

ビルド時に必要なものはStage1で利用するが、Stage ２の最終成果物のみ抽出する。

```
# --- Stage 1: ビルド環境 ---
FROM python:3.11-slim AS builder
WORKDIR /app
# ビルドに必要なツールを一時的に入れる
RUN apt-get update && apt-get install -y gcc python3-dev
COPY requirements.txt .
# ライブラリを特定のフォルダにインストール
RUN pip install --user -r requirements.txt

# --- Stage 2: 実行環境 ---
FROM python:3.11-slim
WORKDIR /app
# ビルドステージで作成したライブラリだけをコピー
COPY --from=builder /root/.local /root/.local
COPY . .
# パスを通す
ENV PATH=/root/.local/bin:$PATH
CMD ["python", "main.py"]
```
