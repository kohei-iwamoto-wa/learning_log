#### Transit Gateway

#### Transit Gateway とは

複数の VPC や オンプレミス環境をつなぐ仮想ルーターのようなサービスです。

#### メリット

VPC ピアリングの場合は、VPC 同士の接続は、1 対 1 の接続しかできないため、 VPC 同士の接続が増えれば増えるほど管理が複雑になります。

#### デメリット

VPC ピアリングに比べて料金が高くなりがち

* 料金
  * データ転送料のみ
* Transit Gateway
  * 時間単位の利用料 + データ処理料

#### 用語

* アタッチメント
  * VPC に Transit Gateway と接続するためのアタッチメントです。
    * VPC アタッチメントには /28 などの小さな CIDR の専用サブネットを推奨
* ルートテーブル
  * Transit Gateway が持つ経路情報
  * １つ以上を作成することができる
* アソシエーション
  * アタッチメントとルートテーブルを関連付ける
* プロパゲーション
  * アタッチメントのルート情報を自動的にTransit Gateway に登録させる

#### Shared VPC（アーキテクチャ）

* 通信させたいVPC と 通信させたくない VPC を route table で分割する。

#### Inspection VPC による Traffic 監査

インタネットへの出入り口を１つに統合とAWS Network firewall でトラフィックの監査を行う。

#### Direct Connect + Site-to-Site VPN 冗⻑構成

Direct Connect と Site To Site VPN を併用することでオンプレミスとの通信を冗長化する。

２つとも有効な場合は、Direct Connect が優先される。
