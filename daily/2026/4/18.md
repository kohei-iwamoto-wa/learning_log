### content addressable storage

Docker のイメージ保管モデルは、コンテンツ・アドレッサブルストレージモデルを利用しています。

docker では、イメージを名前やタグでイメージを識別しているのではなく、ハッシュ値で同一性を識別しています。

* 10個の異なるイメージがすべて同じ「Ubuntu 22.04」をベースにしている場合、Dockerはそのベースレイヤーを1回分しか保存しません
  * 「名前」ではなく「中身のハッシュ」で保存場所を決めるため、同じ内容のデータは1つにまとめられるからです。
* 確実に同一のイメージを使用したい場合は、ハッシュ値を指定するのが良いでしょう。
  * 例）docker pull nigelpoulton/k8sbook@sha256:13dd59a0...bce2e14b

* docker image には、２つのハッシュがある
  * content hash
    * 展開された（非圧縮）状態のレイヤーのハッシュ。
    * push , pull時
  * distribution hash
    * 圧縮された状態のレイヤーのハッシュ。

## Egress-Only インターネットゲートウェイ

nat gateway 不要
ipv6のみ利用可能
名前の通りEgressのみ
ステートフルなので、レスポンスは返ってくる
