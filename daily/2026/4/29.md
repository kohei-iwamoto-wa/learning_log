### 変数（variable）

変数（variable）を設定することができる。

```
variable "example_instance_type" {
  default = "t3.micro"
}

resource "aws_instance" "example" {
  ami           = "ami-0c3fd0f5d33134a76"
  instance_type = var.example_instance_type
}
```

変数（variable）はコマンドで上書き可能です
```
terraform plan -var 'example_instance_type=t3.nano'
```

### ローカル変数（locals）

locals の場合は、コマンド実行時に上書きすることができない。

```
locals {
  example_instance_type = "t3.micro"
}

resource "aws_instance" "example" {
  ami           = "ami-0c3fd0f5d33134a76"
  instance_type = local.example_instance_type
}
```

### Output

terraform apply　実行後、インスタンスID が出力される

```
resource "aws_instance" "example" {
  ami           = "ami-0c3fd0f5d33134a76"
  instance_type = "t3.micro"
}

output "example_instance_id" {
  value = aws_instance.example.id
}
```

### 実務で意識すべきこと

state ファイルは、開発者で管理する必要がある。これを噴出した場合、状態がわからなくなるため、管理方法の設計は行う必要ある

moduleがあるので、cloudformationと比べて、柔軟に構築できる

cloudformationではcfnで作成したリソースを探すのは手間だが、terraformだと、コマンド１つで確認できます。

terraform自体のバージョンアップを行う必要がある
