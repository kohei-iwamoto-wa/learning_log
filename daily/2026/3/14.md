### Namespace（Linux）

#### Namespace とは

Namespace は、Linix上のリソースを独立させるためのものです。

通常Linux上では、プロセスはすべてのリソース（ファイル、ネットワーク、プロセス ID など）を共有していますが、Namespace を作成することでプロセス群を隔離させることができます。

この技術は、Docker などのコンテナ技術の中核をなす要素の１つです。

#### 名前空間の種類

| 名前空間名 | 説明 |
|-----|-----|
| mount | マウントポイント。 |
| uts | ホスト名、NISドメイン　※ 名前空間ごとにホスト名をつけられる |
| ipc | System V IPC, POSIX メッセージキュー  ※ ほかプロセスと通信を遮断|
| pid | プロセスID ※ 他の名前空間のプロセスやホストのプロセスは見えない|
| network | ネットワークデバイス、スタック |
| user | ユーザーおよびグループID |
| Control groups | コントロールグループのroot ディレクトリ |

#### 名前空間を検証してみる

##### localのホスト名を確認

```
sh-4.2$ hostname
localhost
```

##### 新しい名前空間作成（utsの分離）
　
```
sh-4.2$ sudo unshare -u /bin/bash
[root@localhost bin]# hostname newhost
[root@localhost bin]# hostname
newhost
```
※ -u オプションは `uts` を分離する。

##### 新しい名前空間からぬける

```
[root@localhost bin]# exit
exit
```

#### ホスト名を確認する

ローカルのホスト名に戻っている

```
sh-4.2$ hostname
localhost
```

### 学んだこと

コンテナの分離はLinux のnamespace を利用して実現している。

namespace や cgroup を活用すると自作コンテナを作成することが困難だが、Docker を使えば、コマンド１つでコンテナが立ち上がる。
