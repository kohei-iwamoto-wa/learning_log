### cgroup（コントロールグループ）

#### cgroup とは

cgoup は、複数のプロセスを一つにまとめてグループ化します。

そのグループの利用可能なリソースを制限管理することを可能にできます。

これはコンテナ技術の主要な要素の一つとして利用されています。

各コンテナは、完全に独立しているように見えますが、実際は１つの OS 上でプロセスとして存在します。

コンテナがお互いにリソースを奪い合わないようにcgroupでコンテナが利用可能なリソースを設定しておくことができます。

#### cgroup の場所

> ls /sys/fs/cgroup

#### コンテナの利用可能なリソースを確認（メモリ）

```
# コンテナのメモリを512m にして実行する。
docker run -d --name test-cgroup --memory="512m" alpine sleep 10000

# コンテナに入る
docker exec -it test-cgroup /bin/sh

# cgroupの設定を確認
cat /sys/fs/cgroup/memory/memory.limit_in_bytes
536870912s # 512mbで制限されている。
```

#### コンテナの利用可能なリソース（メモリ）を変更

```
sh-4.2$ pwd
# コンテナごとにディレクトリが分かれている。↓コンテナの実態
/sys/fs/cgroup/memory/docker/eb7a7b0f75bca2104e5d15b9782782f748fcebfd13838e98bd64e270f93c4583

# コンテナディレクトリの中身
sh-4.2$ ls /sys/fs/cgroup/memory/docker/eb7a7b0f75bca2104e5d15b9782782f748fcebfd13838e98bd64e270f93c4583
cgroup.clone_children  memory.kmem.failcnt             memory.kmem.tcp.limit_in_bytes      memory.max_usage_in_bytes        memory.move_charge_at_immigrate  memory.stat            tasks
cgroup.event_control   memory.kmem.limit_in_bytes      memory.kmem.tcp.max_usage_in_bytes  memory.memsw.failcnt             memory.numa_stat                 memory.swappiness
cgroup.procs           memory.kmem.max_usage_in_bytes  memory.kmem.tcp.usage_in_bytes      memory.memsw.limit_in_bytes      memory.oom_control               memory.usage_in_bytes
memory.failcnt         memory.kmem.slabinfo            memory.kmem.usage_in_bytes          memory.memsw.max_usage_in_bytes  memory.pressure_level            memory.use_hierarchy
memory.force_empty     memory.kmem.tcp.failcnt         memory.limit_in_bytes               memory.memsw.usage_in_bytes      memory.soft_limit_in_bytes       notify_on_release

# メモリ上限を変更する（ホスト側）
sh-4.2$ echo 268435456 | sudo tee memory.limit_in_bytes
268435456
# コンテナに入る
sh-4.2$ sudo docker exec -it test-cgroup /bin/sh
# メモリの制限を確認すると256mbに変更されている。
/ # cat /sys/fs/cgroup/memory/memory.limit_in_bytes
268435456
```
※ 制限の変更には再起動不要
