### du

du コマンドは、ファイルやディレクトリの容量を出力します。

-s (summary) は、ディレクトリ内の合計値
-h （human-readable）を容量をK, M,G 形式で読みやすく表示する。

```
xxx@yyy:~$ sudo du -sh /*
114G	/var

11G	/snap

7.0G	/usr
4.1G	/swap.img
1.1G	/opt
609M	/logs
217M	/boot
12M	/etc

2.5M	/run

184K	/root
16K	/lost+found
8.0K	/media
4.0K	/srv
4.0K	/sbin.usr-is-merged
4.0K	/mnt

4.0K	/bin.usr-is-merged
0	/sys
0	/sbin
0	/proc
0	/lib64
0	/lib
0	/dev
0	/bin
```

### dh

ディスクパーティションごとの容量を表示する。

-h 容量を人が読みやすい形式で表示する。

```
xxx@yyy:~$ df -h
Filesystem      Size  Used Avail Use% Mounted on
tmpfs           1.6G  2.4M  1.6G   1% /run
/dev/nvme0n1p2  233G  145G   76G  66% /
tmpfs           7.8G  345M  7.4G   5% /dev/shm
tmpfs           5.0M  8.0K  5.0M   1% /run/lock
efivarfs        192K   60K  128K  32% /sys/firmware/efi/efivars
/dev/nvme0n1p1  1.1G  6.2M  1.1G   1% /boot/efi
tmpfs           1.6G  172K  1.6G   1% /run/user/1000
```

### inode

inode とはファイルの属性情報を格納するデータ構造です。

inode 番号は、ファイルの識別子です。OS はファイルをinode 番号で判別します。

ls -i コマンドで inode 番号を 表示することができます。

```
ls -ila
11407730 drwxrwxr-x  8 john-doe john-doe 4096  3月 10 22:26 .
10639122 drwxrwxr-x 22 john-doe john-doe 4096  3月  7 16:06 ..
11407738 drwxrwxr-x  9 john-doe john-doe 4096  3月 18 23:10 .git
11403401 -rw-rw-r--  1 john-doe john-doe    6  3月 10 22:26 .gitignore
11407740 -rw-rw-r--  1 john-doe john-doe  326  3月  7 00:17 README.md
11407732 drwxrwxr-x  2 john-doe john-doe 4096  2月 27 23:05 aws
11408194 drwxrwxr-x  4 john-doe john-doe 4096  3月 10 22:25 daily
11407736 drwxrwxr-x  2 john-doe john-doe 4096  3月 11 23:07 kubernetes
11407737 drwxrwxr-x  2 john-doe john-doe 4096  3月 10 22:27 linux
11554212 drwxrwxr-x  2 john-doe john-doe 4096  3月 10 22:26 tmp
```

#### inode 番号は有限

inode 番号は、有限のため、ファイル数が増えると inode 番号が枯渇します。

枯渇すると、新たにファイルを作成することができなくなります。

#### inode 利用率調べる

df の -i オプションを利用する。

```
df -i
Filesystem       Inodes  IUsed    IFree IUse% Mounted on
tmpfs           2026977   1467  2025510    1% /run
/dev/nvme0n1p2 15564800 916673 14648127    6% /
tmpfs           2026977    415  2026562    1% /dev/shm
tmpfs           2026977      4  2026973    1% /run/lock
efivarfs              0      0        0     - /sys/firmware/efi/efivars
/dev/nvme0n1p1        0      0        0     - /boot/efi
tmpfs            405395    208   405187    1% /run/user/1000
```

#### よくある原因

ログの分割が細かすぎる

一時ファイルが削除されず大量に残っている
