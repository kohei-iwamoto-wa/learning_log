### Pod identity webhook（EKS）

#### Pod identity webhook とは

Pod が AWS サービスにアクセスする際に安全簡単に認証を行う仕組み

#### 仕組み

1. Pod 作成要求
2. Pod作成要求を webhook が検知
3. webhook が Pod に環境変数を注入する
   1. AWS_ROLE_ARN：使用するIAMロールの識別子
   2. AWS_WEB_IDENTITY_TOKEN_FILE：一時的な認証トークン（Web Identity Token）のパス

### logrotate 

logrotate は、週次や日時といった周期やログファイルのサイズが一定サイズ以上になるとログファイルを分割するといった機能があります。

また、古いログファイルは、gzipにするなどログファイルの圧縮をするといった設定やログの削除設定を入れることができます。

そうすることによってログファイルによるディスクの圧迫対策を行うことでできる機能です。

#### logrotate の設定

以下にlogrotate の設定があります。
※ Amazon Linux2の場合

> /etc/logrotate.conf

#### logrotate.conf の内容

```
# see "man logrotate" for details
# rotate log files weekly
weekly # ログローテーションの周期

# keep 4 weeks worth of backlogs
rotate 4 # 4週間分のログファイルを保持します

# create new (empty) log files after rotating old ones
create # 古いログファイルをログローテーションしたら新しいログファイルを生成する

# use date as a suffix of the rotated file
dateext # ログファイルのサフィックスに日付情報を付与する

# uncomment this if you want your log files compressed
#compress　# compressでgzip化を有効にします

# RPM packages drop log rotation information into this directory
include /etc/logrotate.d

# no packages own wtmp and btmp -- we'll rotate them here
/var/log/wtmp {
    monthly
    create 0664 root utmp
        minsize 1M
    rotate 1
}

/var/log/btmp {
    missingok
    monthly
    create 0600 root utmp
    rotate 1
}

# system-specific logs may be also be configured here.
```

ログファイルがディスクを圧迫する場合、compressでファイルを圧縮すると容量に空きがでる

それでもディスク容量が開かない場合は、削除対応やクラウドに送るなど対応が必要になる。

#### ログローテーションのデバッグモード

> sudo logrotate -d /etc/logrotate.conf

#### ログローテーション強制実行

> sudo logrotate -f /etc/logrotate.conf
