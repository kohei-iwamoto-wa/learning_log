### liveness probe

コンテナの健康状態を定期的にチェックし、unhealthy であれば、コンテナを再起動する。

#### liveness probe の設定

* livenessProbe の設定した manifest を適用する。

```
apiVersion: v1
kind: Pod
metadata:
  labels:
    test: liveness
  name: liveness-exec
spec:
  containers:
  - name: liveness
    image: nginx
    args:
    - /bin/sh
    - -c
    # /tmp/healthy を作成、30秒した後、/tmp/healthy を削除する。
    # livenessProbe テスト用のコマンド
    - touch /tmp/healthy; sleep 30; rm -f /tmp/healthy; sleep 600
    livenessProbe:
      # cat /tmp/healthy が成功すれば、healthy です
      exec:
        command:
        - cat
        - /tmp/healthy
      # ヘルスチェック開始までの時間
      initialDelaySeconds: 5
      # 5秒毎にヘルスチェックする
      periodSeconds: 5
```

* pod の状況確認（問題なし）

```
kubectl get pod -n default
NAME            READY   STATUS    RESTARTS   AGE
liveness-exec   1/1     Running   0          30s
```
* pod の状況確認（リスタートしてる）
  * コンテナに問題が生じたら再起動する、これは、kubernetes オートヒーリングの機能

```
kubectl get pod -n default
NAME            READY   STATUS    RESTARTS      AGE
liveness-exec   1/1     Running   1 (29s ago)   109s
```

### readiness probe

Pod の 健康状態を定期的にチェックし、Ready でなければ、Service から Pod を切り離す。

準備ができていない pod にアクセスできないようにできる。

※ Pod が準備できている状態の定義は、Pod 内のコンテナがすべて準備が整った状態です。

#### readiness probe の設定

```
apiVersion: v1
kind: Pod
metadata:
  labels:
    test: liveness
  name: liveness-exec
spec:
  containers:
  - name: liveness
    image: registry.k8s.io/busybox
    args:
    - /bin/sh
    - -c
    - touch /tmp/healthy; sleep 30; rm -f /tmp/healthy; sleep 600
readinessProbe:
  exec:
    command:
    - cat
    - /tmp/healthy
  initialDelaySeconds: 5
  periodSeconds: 5
```
※ livenessProbe と構成は同じ

### 学び

Kubernetes の livenessProbe は、コンテナの生存確認用で生存していなければ、再起動を行う。

ReadinessProbe は、コンテナの準備確認を行い、準備ができていなければ、サービスから pod を切り離す。
