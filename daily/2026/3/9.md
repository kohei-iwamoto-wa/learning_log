### OOMKiller（Out Of Memory Killer）

#### OOMKiller とは

OOMKiller（Out Of Memory Killer）とは、メモリが枯渇した場合、システム全体が停止しないためにメモリ消費量が多いプロセスを自動的に停止する機能です。

#### Out Of Memory を再現する


stress コマンドを利用することでOut Of Memoryの状況を再現する。

```
sh-4.2$ stress --vm 50 --vm-bytes 700M --timeout 120s

stress: WARN: [30126] (417) now reaping child worker processes
```

#### Out Of Memory が生じたか確認する

```
sh-4.2$ dmesg | grep -i "killed process"
[100847.231391] Out of memory: Killed process 22049 (stress) total-vm:724388kB, anon-rss:38864kB, file-rss:0kB, shmem-rss:0kB, UID:1001 pgtables:140kB oom_score_adj:0
[100848.481072] Out of memory: Killed process 22046 (stress) total-vm:724388kB, anon-rss:42548kB, file-rss:0kB, shmem-rss:0kB, UID:1001 pgtables:148kB oom_score_adj:0
[100849.871613] Out of memory: Killed process 22030 (stress) total-vm:724388kB, anon-rss:30928kB, file-rss:0kB, shmem-rss:0kB, UID:1001 pgtables:124kB oom_score_adj:0
```

 `dmesg` は、今回のようにメモリや I/Oエラーなどハードウェアやドライバに関する不具合調査する際に使用する。

#### OOMKiller の優先順位

OOMKiller で Killされるプロセスの優先順位を設定しておくことができる。

Killされるとシステムに重大な障害が起こるプロセスの優先順位を下げるように設定する。

```
cat  /proc/{PID}/oom_score_abj
0
```

oom_score_abj は、0~1000まで設定することができ、-1000は優先順位が低く、1000が優先順位が高いため、Kill されたくないプロセスは -1000 を設定する。syuu
