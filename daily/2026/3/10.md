### systemd 

#### systemd とは

systemd は、LinuxなどのUnix系のシステムを行動する際にプロセスを管理するプログラムのことです。

#### systemd 起動のながれ

1. 物理サーバーの電源が起動
2. BIOS が起動
3. BIOS がブートローダーを起動
4. ブートローダーが OS のカーネルを起動
5. カーネルの起動時に systemd が起動する
6. systemd が全サービス（nginx等）を起動する

#### systemd のユニットとは

ユニットとは、systemd の管理単位のことです。

* .service
  * 実行可能なプログラムやデーモン
* .target
  * 複数のユニットをまとめる。
* .mount
  * ファイルシステムのマウントについて管理
* .socket
  * ソケットファイルの管理
* .device
  * デバイスファイルを管理
* timer
  * 定期的なタスク実行の管理
  
#### ユニットのメリット

* 管理方法が一元管理
* 依存関係の管理
* 起動順序の制御

#### ユニットファイルの場所

> /etc/systemd/system/

#### ユニットファイルを開く

ユニットファイルを開くと設定が書かれています。

以下はbluetooth サービスです。

```
 cat /etc/systemd/system/dbus-org.bluez.service

# ユニットセクション
[Unit]
# サービスの説明
Description=Bluetooth service
# マニュアル
Documentation=man:bluetoothd(8)
# ディレクトリが存在しているときだけ起動
ConditionPathIsDirectory=/sys/class/bluetooth

# サービスセクション
[Service]
Type=dbus
BusName=org.bluez
# このディレクトリのサービスを起動する
ExecStart=/usr/libexec/bluetooth/bluetoothd
NotifyAccess=main
#WatchdogSec=10
# 異常終了した場合に再起動し
Restart=on-failure
CapabilityBoundingSet=CAP_NET_ADMIN CAP_NET_BIND_SERVICE
# このプロセスは一つだけ起動できる
LimitNPROC=1

# Filesystem lockdown
ProtectHome=true
ProtectSystem=strict
PrivateTmp=true
ProtectKernelTunables=true
ProtectControlGroups=true
StateDirectory=bluetooth
StateDirectoryMode=0700
ConfigurationDirectory=bluetooth
ConfigurationDirectoryMode=0555

# Execute Mappings
MemoryDenyWriteExecute=true

# Privilege escalation
NoNewPrivileges=true

# Real-time
RestrictRealtime=true

# インストールセクション
[Install]
WantedBy=bluetooth.target
Alias=dbus-org.bluez.service

```
